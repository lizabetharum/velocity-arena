Before You Start
  • Cardboard 15 × 20 cm — 1 per team + 2 spare sheets each
  • Floor lanes taped (same marks as morning — bot nose at 40 cm, ball mark separate)
  • Scissors, painter's tape, metric ruler at each table
  • MakeCode template open: Button A → forward [X]% 3500ms → stop
  • Power reference table displayed where all teams can see it
  • All bots charged and Bluetooth-paired before teams arrive
  • makecode.microbit.org/S34942-60930-12957-82725
Students Produce
  • Cardboard scoop attached to bot
  • Data dashboard: 3 Power settings × 3 shots + averages
  • Observation sentence: "Quadrupling charge speed [did / did not] quadruple kick distance, because…"

Timing
0–2Launch "Build a scoop. It has to hit the ball flat — not ride over it, not push it sideways, not block the sensor. 20 minutes. Go." Walk away immediately.
2–18Build Circulate silently — don't suggest shapes. At 8 min if stuck: "Does it hit the ball flat?" At 18 min: "Two minutes. Attach it. Get to your lane."
20Ph.2 Launch "Choose Power. Calculate charge speed. Write both on worksheet before opening MakeCode." Wait for paper before code.
22–373 Shots Nose on Tape 1, ball on Tape 2 — every shot. Measure start to stop. Retrieve and reset between shots.
37–40Average "Average 3 shots. Record on dashboard. Don't pick Power 2 until the average is written."
40–57Power 2 Calculate → write → code → 3 shots → average. Watch for P values too close. "Is that gap big enough to see a pattern?"
57–62Tweak "Look at your scoop. Did the ball veer? One adjustment only." No full rebuilds.
62–72Power 3 Calculate → write → code → 3 shots → average. Fill final row. "Last shot. Measure it. Bots stop."
75–79Find Ratio "Find the pair where charge speed quadrupled. Did kick distance also quadruple?" 3 min quiet.
79–88Sentence "Quadrupling charge speed [did / did not] quadruple kick distance, because…" The because is the point.
88–90Hard Stop Scoop stays on bot. Bots to charging stations. Dashboard complete with 3 settings + observation sentence.

Watch For → Redirect
Skips paper calc "Paper before code — every setting, every time."
Ball veers sideways Scoop geometry problem. Don't change Power. "Does it hit the ball flat?"
3rd rebuild at 60 min Intervene directly. "Pick the one giving the most consistent shots. You need data now."
Values too close (P4/P5) "What ratio are you testing? Is that gap big enough to see a pattern?"
Blank "because" field "What did charge speed do? What did distance do? What's different about those two things?"
Bot stops dead on contact Scoop absorbing impact. Check tape attachment — face may need to be more vertical.
Hard Stop Data dashboard complete with 3 Power settings and averages. Observation sentence written. Bots on charging stations.
