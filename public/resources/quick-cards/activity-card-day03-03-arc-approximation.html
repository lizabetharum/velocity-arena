Before You Start
  • Floor grid set up — origin, axis labels, ball at (4,2), corners (0,0) (8,0) (0,4) (8,4)
  • Distance formula on board — leave up all session
  • Arc Trace worksheets on tables — one per team
  • Stat cards out — Speed + Endurance already filled
  • 3 colors of dot stickers per team (coverage challenge)
  • Laptops open to MakeCode
  • Do not erase the board between sections — the cumulative board is the block's visual record.
Students Produce
  • Arc Trace Sheet — 3 paths drawn, one per TURNING value, different colors
  • Coverage trial square counts recorded for each TURNING value
  • Turning value written on stat card

Timing — 7 Parts, 90 min
0:20Launch SequenceTeams pick corner → calculate distance to ball (4,2) → program moveToCoordinate → run, mark error. Class compare: shortest path ≠ best approach angle. "Opening move done. Match doesn't pause after contact."
20:28Polygon WalkPairs: robot + programmer. R1: arc in 2 turns (90° each) — chunky L. R2: same arc in 8 smaller turns — smoother. Board: MORE SEGMENTS ≠ TRUE CURVE / MORE SEGMENTS = BETTER APPROXIMATION.
28:38Arc Trace SheetsDraw 3 paths on grid for T=2, 5, 10. Different color each. More segments + more deflection per segment for higher T. Stop-check at 33 min: "Which T made widest arc? Tightest?" Board: COVERAGE vs. CONCENTRATION.
38:40BridgeBoard: "Morning: WHERE does the robot go? Afternoon: WHAT AREA does it cover?" TURNING controls the second question.
40:55Differential Code3 configs: (50,50) straight; (35,65) T=5; (30,70) T=10. Predict endpoint → sticker → run → compare. Then 5× pwmCruiseControl(35,65) 500ms loop — predict arc, run, compare.
55:70Coverage ChallengeFull arena from (4,2). 3 trials × 15 sec, one per T value. Sticker every square touched. Analyst counts. Round-robin results on board.
70:80Stat CardWrite Turning on stat card. Sentence starters on board. 3–4 whole-room shares — listen for strategy language.
85:90Bridge to Power"Tomorrow your Fabricator builds something on the bot. Arc path × hitting force = coverage strategy." Hard stop 90 min.
Say This
Arc truth
"Was round 2 actually a curve?" Pause. "No. Still straight lines — we used more of them. That is arc approximation."
Wheels
"Right wheel faster → robot curves left. turnRatio controls the wheel speed difference — which sets the radius."
Coverage
"No single correct answer. It depends on where you start and how long you run. Design decision — not a math problem with one solution."
Bridge
"Arc path times hitting force equals coverage strategy. Tomorrow you add the last number."
Not this
Don't say higher T = better coverage. Don't confirm arcs are curves — push back every time. Don't erase the board between sections.
Watch For → Redirect
"Higher T always means more coverage""What happens at T=15?" High T spirals tight — concentrates rather than sweeps.
Arc Trace: parallel lines instead of curving path"After each segment your direction changes. Show that — the path should bend as it goes."
Bot drifts inconsistently in coverage challenge"Analyst: note it. Real arena bots drift. Does that change your T strategy?" Hardware variation is data.
Behind at 55-min markCut Config 3 from differential code, go straight to multi-segment loop. Coverage challenge has more learning weight.
Hard Stop Turning value written on stat card and every team completes at least 2 of 3 coverage trials before dismissal.
