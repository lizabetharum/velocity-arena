The Endurance Formula
Understand the decay — then predict what your bot will do
Power Decay Formula
P(t) = 100 − (20 − E) × 1.5 × t
E = your Endurance stat  ·  t = time in minutes  ·  P(t) = motor % at that moment
Key Terms
P(t)Motor power remaining — starts at 100%, drops over time
Endurance (E)Your stat allocation (1–20). Higher E = slower decay = more consistent bot
decay rate20 − E. How fast your power drains each minute
tTime in minutes (0 to 3 for a full match)

⚠️ Two Problems — Before You Can Code This

The formula is mathematically correct. But MakeCode can't run it as written. Two things need to change.

Problem 1 — The Decimal (1.5)

MakeCode silently drops all decimals. When you type 1.5, the micro:bit reads 1. No error. No warning. Your decay rate gets cut in half and you'll never know why.

Fix: Replace 1.5 with 3 ÷ 2. Same math — but now MakeCode multiplies by 3 first, then divides by 2. No decimals anywhere.
Problem 2 — Time in Minutes

30 seconds is 0.5 minutes. MakeCode reads 0.5 as 0. So t = 0 every single loop. Zero times anything is zero — no decay ever happens. Your bot runs at full power the whole time.

Fix: Measure time in seconds. MakeCode gives you time in milliseconds — divide by 1000 to get whole seconds.
Transforming the Formula

Same prediction. New form that the micro:bit can actually calculate.

1
Original
P(t) = 100 − (20 − E) × 1.5 × t
Correct math — broken for the computer (decimal, time in minutes)
2
Replace 1.5 with 3÷2
P(t) = 100 − (20 − E) × 3 × t ÷ 2
Mathematically identical — no decimal anywhere
3
Replace t (minutes) with t÷60 (seconds)
P(t) = 100 − (20 − E) × 3 × (t ÷ 60) ÷ 2
Now t is in seconds — a whole number the computer can use
4
Collapse the division: 3 ÷ 2 ÷ 60 = 1 ÷ 40
P(t) = 100 − (20 − E) × t ÷ 40
Final form. Prediction unchanged — just a form the computer can run.

Key Idea
Speed controls how fast. Endurance controls how long. Every bot starts at 100% power — your Speed sets the top speed, your Endurance sets how quickly it drains.
Program & Test
Build the decay code in MakeCode — then run your bot
Read this before you start.
The bot runs for exactly 2 seconds each time you press A. Button B sets which moment in the match you're simulating. You'll run four 2-second bursts — at t = 0, 1 min, 2 min, and 3 min — and mark where the bot stops each time. The formula predicts the bot should slow down. You're here to see if it does.
The Code Structure

Three blocks work together. You set your stats in on start, use Button B to choose the match time you want to test, then press Button A to drive for 2 seconds at that power level.

on start — set your stats
set SPEED to 0   // ← change to your stat card value
set ENDURANCE to 0   // ← change to your stat card value
set TEST_DURATION_MS to 2000   // 2-second burst
set simulatedMatchTime to 0   // starts at beginning of match
computeStats — formula runs here
set basePower to SPEED × 100 ÷ 20   // top wheel speed
set DECAY_RATE to 20 − ENDURANCE
set endurancePercent to 100 − DECAY_RATE × simulatedMatchTime ÷ 40
set endurancePercent to max of endurancePercent and 20   // 20% floor
set currentPower to round(basePower × endurancePercent ÷ 100)
on button A — run 2-second burst
call computeStats   // calculate power for current simulatedMatchTime
set left/right wheel speed to currentPower %
while running and elapsed < TEST_DURATION_MS:
  // check border — handle if hit
  pause BORDER_CHECK_MS
stop all wheels  →  show string "DONE"
on button B — cycle simulated match time
if simulatedMatchTime = 0:  set to 60   // LED: 1 column
else if simulatedMatchTime = 60:  set to 120   // LED: 2 columns
else if simulatedMatchTime = 120:  set to 180   // LED: 3 columns
else:  set to 0   // LED: center dot
simulatedMatchTime is the key variable. It feeds into computeStats, which recalculates how much power the bot should have at that point in the match. You change it with B — not by waiting for time to pass.

📏 How the Test Works

You run four 2-second bursts — one for each simulated match time. Each burst starts from the same line. Tape marks where the bot stops. You measure all four at the end.

Burst 1
t = 0 · Start
Center dot LED. Bot at full power. Press A.
Burst 2
t = 60 s · 1 min
1-column LED. Press B once, then A.
Burst 3
t = 120 s · 2 min
2-column LED. Press B again, then A.
Burst 4
t = 180 s · 3 min
3-column LED. Press B again, then A.
📏 How to Measure
  1. Put a piece of tape on the floor — this is the starting line. It stays there for all four runs.
  2. After each 2-second run, place a piece of tape at the front of the bot to mark where it stopped.
  3. Move the bot back to the starting line before each new run.
  4. After all four runs, measure from the starting tape to each mark. Record in centimeters (cm).
▶ Run the Test — Step by Step
  1. 1
    Put a strip of tape on the floor. This is the starting line for all four runs.
  2. 2
    Place the bot at the starting line. Check the LED — it should show the center dot (t = 0). If it shows columns, press B until you're back to the center dot.
  3. 3
    Press A. The bot drives for 2 seconds and stops. Place a piece of tape at the front of the bot to mark where it stopped. This is your Burst 1 mark.
  4. 4
    Move the bot back to the starting line.
  5. 5
    Press B. The LED shows 1 column — the bot is now simulating t = 60 seconds (1 minute into the match).
  6. 6
    Press A. Mark where the bot stops. This is your Burst 2 mark.
  7. 7
    Move the bot back to start. Press B → LED shows 2 columns (t = 120 s, 2 min). Press A. Mark where it stops — Burst 3.
  8. 8
    Move the bot back to start. Press B → LED shows 3 columns (t = 180 s, 3 min). Press A. Mark where it stops — Burst 4.
  9. 9
    Measure from the starting tape to each of the four marks. Record all distances in centimeters.
Emergency stop: Press A + B together at any time. The bot stops immediately and shows an X on the LED.

The Gap Sentence
Prediction + observation + gap = a complete scientific statement. The formula predicts a perfect machine. Your bot is not a perfect machine. Name one real-world cause of your gap — that's how models improve.
Watch Out — Common Errors
Pressed B too many times
B cycles 0 → 60 → 120 → 180 → 0. Check the LED pattern before pressing A. If you're on the wrong time, keep pressing B until you cycle back.
SPEED or ENDURANCE still set to 0
If the bot doesn't move, check on start. Both values default to 0 in the starter — you must type yours in.
Measuring before the bot fully stops
Wait for "DONE" to appear on the LED screen before placing your tape mark. The bot stops automatically at 2 seconds.
Bot hit the border and turned before stopping
The border detection code will turn the bot away from the edge. If this happens, the distance isn't straight-line. Re-run that burst with the bot positioned away from any border.
