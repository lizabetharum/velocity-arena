How the Code Works
// ── on start — runs once when micro:bit turns on ──
on start:
  set SPEED = 6                    // ← change to 12 for Run 2
  set motorPower = SPEED × 100 ÷ 20  // = 30% for Run 1
// ── Button A — GO ─────────────────────────────────
on button A pressed:
  show string "Go"
  clear screen
  set left wheel motorPower%, right wheel motorPower%
  pause (5000 ms)           // wheels spin for 5 seconds
  stop all wheels
// ── Button B — EMERGENCY STOP ─────────────────────
on button B pressed:
  stop all wheels
  show leds  // X pattern
  pause (3000 ms)
  clear screen

Between runs, you only change one thing: SPEED = 6 for Run 1 → SPEED = 12 for Run 2. Download again after changing it.

The MakeCode Screen — What’s Where
What Each Block Does
on start
Basic (blue). Runs once when the micro:bit turns on. Put your starting values here — SPEED and motorPower get set up before any button is pressed.
on button A pressed
Input (magenta). Everything inside runs when someone presses button A. This is your GO button — starts the drive sequence.
set SPEED to 6
Variables (red). Creates a named box called SPEED and puts 6 inside it. Change this to 12 for Run 2 — that's the only thing you change between runs.
set motorPower to SPEED×100÷20
Variables (red). Your formula in block form. The micro:bit computes SPEED×100÷20 and stores the result. SPEED=6 → 30. SPEED=12 → 60.
show string "Go"
Basic (blue). Scrolls the word "Go" across the micro:bit LED screen — a visual cue that the bot is about to move.
set left wheel speed motorPower %,
right wheel speed motorPower %
Cutebot Pro (green). Spins both wheels at motorPower %. This is how the bot drives forward — you set each wheel to the same value so it goes straight.
pause (ms) 5000
Basic (blue). Waits 5,000 milliseconds = 5 seconds. The wheels keep spinning during this pause. After 5 seconds the next block runs.
set all wheel to stop immediately
Cutebot Pro (green). Cuts power to both wheels instantly. Without this the bot would keep rolling after the pause ends.
on button B pressed
Input (magenta). Emergency stop. If anything goes wrong, press B. Stops all wheels and shows an X on the LED.
Step-by-Step — Build Your Program
Phase 1 — Open MakeCode & Add the Extension
  1. 1
    Go to makecode.microbit.org. Click New Project and give it a name like "Speed Test." 💡 Sign in first — it saves your work automatically.
  2. 2
    You'll see an on start block and a forever block. Delete the forever block — drag it to the trash or press Delete. Keep on start.
  3. 3
    Scroll down the categories and click Advanced, then Extensions. Search Cutebot Pro and click it. A new Cutebot Pro category appears.
Phase 2 — Set Up Variables in "on start"
  1. 4
    Click Variables (red). Click "Make a Variable." Name it SPEED. Click OK.
  2. 5
    From Variables, drag set SPEED to 0 inside the on start block. Click the 0 and type 6.
  3. 6
    Back in Variables, click "Make a Variable" again. Name it motorPower. Click OK.
  4. 7
    Drag set motorPower to 0 into on start, below the SPEED block.
  5. 8
    Click Math (purple). Drag a 0 × 0 block into the motorPower slot. Then nest a 0 ÷ 0 block so you get: (SPEED × 100) ÷ 20. Pull the SPEED oval from Variables and drop it as the first number.
Phase 3 — Build Button A (Drive Forward)
  1. 9
    Click Input (magenta). Find on button A pressed and drag it into the workspace — separate from on start.
  2. 10
    From Basic (blue), drag show string "" inside button A. Click the empty string and type Go.
  3. 11
    From Basic, drag clear screen inside button A, below show string.
  4. 12
    Click Cutebot Pro (green). Find set left wheel speed (50)%, right wheel speed (50)%. Drag it inside button A. Replace both 50s with the motorPower variable from Variables.
  5. 13
    Click Basic. Drag pause (ms) 100 inside button A after the wheel block. Change 100 to 5000.
  6. 14
    From Cutebot Pro, drag set all wheel to stop immediately inside button A, below the pause.
Phase 4 — Build Button B (Emergency Stop)
  1. 15
    Click Input. Drag another on button A pressed block. Click the dropdown "A" and change it to B.
  2. 16
    From Cutebot Pro, drag set all wheel to stop immediately inside button B.
  3. 17
    From Basic, drag show leds inside button B. Click the grid to light an X pattern (corners + center).
  4. 18
    From Basic, drag pause (ms) 100 inside button B after show leds. Change to 3000. Drag clear screen below it.

How to Measure Each Run
  1. Put a piece of tape on the floor at the starting line. Start the bot with its front edge at the tape.
  2. Press Button B to cycle to your target speed. Wait for the bot to complete its run (the X appears).
  3. Place your finger at the front of the bot — don't let it move. Measure from the tape to your finger.
  4. Record in centimeters (cm) in your worksheet. Repeat for each of your 3 speeds.

The Big Question

When you doubled Speed (6 → 12), did the distance also double?

Exactly right. A clean 2× ratio confirms the linear relationship. In Velocity Arena, you can predict distance from Speed points without guessing — the formula works.
Take another look. Speed doubled (6 → 12) and motor power doubled (30% → 60%), so distance should roughly double too. Check your ratio in Section 3. If it's between 1.8 and 2.2, the formula still holds — real bots have small variations.
Good observation. The formula predicts a perfect 2×, but real robots have friction, battery differences, and timing variation. A ratio between 1.8× and 2.2× still confirms the linear relationship. The small gap from 2.0 is normal.

💡 Hint for Question 3
Use the formula: motorPower = Speed × 100 ÷ 20.
Speed 12 → 12 × 100 ÷ 20 = 60%.
Speed 16 → 16 × 100 ÷ 20 = 80%.
That's 20% more power. Since the relationship is linear, expect about 20% more distance too.
