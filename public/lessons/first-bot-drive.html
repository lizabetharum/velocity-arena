The Speed Formula
Speed points control how hard the motors run
Today's Formula
Motor Power % = Speed Points × 100 ÷ 20
You have 20 stat points total. Every point you put into Speed is worth exactly 5% motor power.
📖 Key Words
Motor Power % How hard the motors run — like a gas pedal from 0% to 100%
Speed points The number you choose for your bot's Speed stat (0–20)
Linear Same increase every step — double the input, double the output

📖 New Words — Block Coding
MakeCode A free website where you build programs by snapping colored blocks together — no typing needed
Block One instruction the computer follows. Blocks snap together like puzzle pieces.
Variable A named box that stores a number. Like writing "SPEED = 6" on a sticky note.
Event "When something happens, do this." Pressing button A is an event.
Download Sending your finished program from the website to the micro:bit chip
How the Code Works
// ── on start — runs once when micro:bit turns on ──
on start:
  set SPEED = 6                    // ← change to 12 for Run 2
  set motorPower = SPEED × 100 ÷ 20  // = 30 for Run 1
// ── Button A — GO ─────────────────────────────
on button A pressed:
  show string "Go"
  clear screen
  set left wheel speed motorPower%, right wheel speed motorPower%
  pause (5000 ms)               // wheels spin for 5 seconds
  set all wheel to stop immediately
// ── Button B — EMERGENCY STOP ─────────────────
on button B pressed:
  set all wheel to stop immediately
  show leds  // X pattern on LED screen
  pause (3000 ms)
  clear screen

Between runs, you only change one thing: SPEED = 6 for Run 1 → SPEED = 12 for Run 2. Download again after changing it.

① Simulator — virtual bot to test before downloading
② Categories — colored drawers full of blocks
③ Workspace — where you snap your program together
🧩 What Each Block Does
on start
Startup block (Basic — blue). Runs once as soon as the micro:bit turns on. Use it to set your starting values — SPEED and motorPower go here so they're ready before you press any button.
on button A pressed
Trigger block (Input — magenta). Everything inside runs when someone presses button A. This is your "GO" button — it starts the drive sequence.
set SPEED to 6
Variable block (Variables — red). Creates a named storage box called SPEED and puts 6 inside it. Change this number to 12 for Run 2 — that's the only thing you change between runs.
set motorPower to SPEED×100÷20
Formula block (Variables — red). Your Speed formula in block form. The micro:bit does the math and stores the answer in motorPower. SPEED=6 → 30. SPEED=12 → 60.
show string "Go"
Display block (Basic — blue). Scrolls the word "Go" across the micro:bit LED screen — a visual cue that the bot is starting.
set left wheel speed motorPower %,
right wheel speed motorPower %
Drive block (Cutebot Pro — teal). Sets both wheels to spin at motorPower %. This is how the bot drives forward — you control each wheel separately so they run at the same speed.
pause (ms) 5000
Timer block (Basic — blue). Makes the program wait 5,000 milliseconds = 5 seconds. The wheels keep spinning during this pause. After 5 seconds, the next block runs.
set all wheel to stop immediately
Stop block (Cutebot Pro — teal). Cuts power to both wheels instantly. Without this, the bot would keep rolling after the pause ends.
on button B pressed
Emergency stop (Input — magenta). If anything goes wrong, press B immediately. This stops both wheels and shows an X on the LED display.
Step-by-Step — Build Your Program
Phase 1 — Open MakeCode & Add the Extension
  1. 1
    Go to makecode.microbit.org. Click New Project and give it a name like "Speed Test." 💡 Sign in first — it saves your work automatically.
  2. 2
    You'll see an on start block and a forever block. Delete the forever block — drag it to the trash bin on the left, or click it and press Delete on your keyboard. Keep on start.
  3. 3
    Scroll down the categories list and click Advanced, then Extensions. Type Cutebot Pro in the search box and click the result. A new Cutebot Pro category appears in the list.
Phase 2 — Set Up Variables in "on start"
  1. 4
    Click Variables (red). Click "Make a Variable." Name it SPEED. Click OK.
  2. 5
    From Variables, drag set SPEED to 0 inside the on start block. Change the 0 to 6.
  3. 6
    Back in Variables, click "Make a Variable" again. Name it motorPower. Click OK.
  4. 7
    Drag set motorPower to 0 into on start, below the SPEED block.
  5. 8
    Click Math (purple). Drag a 0 × 0 block into the motorPower slot to replace the 0. Then drag a 0 ÷ 0 block and nest it so you get: (SPEED × 100) ÷ 20. Pull the SPEED oval from Variables and drop it as the first number.
Phase 3 — Build Button A (Drive Forward)
  1. 9
    Click Input (magenta). Find on button A pressed and drag it into the workspace — separate from the on start block.
  2. 10
    From Basic (blue), drag show string "" inside button A. Click the empty string and type Go.
  3. 11
    From Basic, drag clear screen inside button A, below show string.
  4. 12
    Click Cutebot Pro (teal). Find set left wheel speed (50)%, right wheel speed (50)%. Drag it inside button A. Replace both 50s with the motorPower variable from Variables.
  5. 13
    Click Basic (blue). Drag pause (ms) 100 inside button A, after the wheel speed block. Change 100 to 5000 (= 5 seconds).
  6. 14
    From Cutebot Pro, drag set all wheel to stop immediately inside button A, below the pause block.
Phase 4 — Build Button B (Emergency Stop)
  1. 15
    Click Input. Drag another on button A pressed block. Click the dropdown "A" and change it to B.
  2. 16
    From Cutebot Pro, drag set all wheel to stop immediately inside button B.
  3. 17
    From Basic, drag show leds inside button B. Click the grid to light up an X pattern (corners + center).
  4. 18
    Click Basic (blue). Drag pause (ms) 100 inside button B, after the show leds block. Change to 3000. Then drag clear screen from Basic below it.
Phase 5 — Download & Test
Click this button in the top-right corner of MakeCode:
⬇ Download
···
  1. 19
    Plug your micro:bit v2 into the computer with the USB cable. It will appear as a drive (like a USB stick).
  2. 20
    Click the ⬇ Download button (top right, purple). You can download a .hex file to your computer, or pair directly with the Cutebot Pro and upload it wirelessly.
  3. 21
    Click the ··· (three dots) next to the Download button and select Disconnect. This safely ends the pairing session before you unplug.
  4. 22
    Gently disconnect the USB cable from the micro:bit v2. Hold the micro:bit v2 between your fingers and carefully pull the cable straight out — do not twist or yank. You do not want to break the connector off the board.
  5. 23
    Place the bot at the starting tape. Press A — you should see "Go" scroll on the LEDs, then the bot drives for 30 seconds and stops.
  6. 24
    Measure the distance. Fill in sections 2 and 3 of your worksheet. Then go back to MakeCode, change set SPEED to 6 → set SPEED to 12, click ⬇ Download again, and do Run 2.
  7. 25
    Answer the Reflect questions in your worksheet.
📏 How do I measure?
  1. Put a piece of tape on the floor at the starting line.
  2. After the bot stops, place your finger at the front of the bot — don't let it move.
  3. Measure from the tape to your finger using a tape measure.
  4. Record your distance in centimeters (cm) on your worksheet.

The Big Question

When you doubled Speed (6 → 12), did the distance also double?

Exactly right. A clean 2× ratio confirms the linear relationship. In Velocity Arena, you can predict distance from Speed points without guessing — the formula works.
Take another look. Speed doubled (6 → 12) and motor power doubled (30% → 60%), so distance should roughly double too. Check your ratio in Section 3. If it's between 1.8 and 2.2, the formula still holds — real bots have small variations.
Good observation. The formula predicts a perfect 2×, but real robots have friction, battery differences, and timing variation. A ratio between 1.8× and 2.2× still confirms the linear relationship. The small gap from 2.0 is normal.

💡 Hint for Question 3
Use the formula: motorPower = Speed × 100 ÷ 20.
Speed 12 → 12 × 100 ÷ 20 = 60%.
Speed 16 → 16 × 100 ÷ 20 = 80%.
That's 20% more power. Since the relationship is linear, expect about 20% more distance too.
