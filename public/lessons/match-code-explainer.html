Your Job

You control two zones. Everything else is automatic.

The match code template comes pre-built. You set your character's stats and tune the launch sequence. The engine handles the rest — roaming the ring, detecting the ball, charging, and recovering from border hits.

⚙️ Zone A — Stat Allocation

Set your four stats

These four numbers control everything: how fast you roam, how quickly your power fades, how sharply you arc, and how hard you hit the ball.

📌Each stat must be at least 1 and no more than 17
📌All four must add up to exactly 20 — press A and see ERR if they don't
📌Set once in on start — they don't change mid-match
🚀 Zone B — Launch Sequence

Tune your opening move

After Button A starts the match, a pre-built 4-step launch sequence drives your bot into the arena before the main loop begins.

📌The default sequence is pre-built — turn left 90°, drive forward, turn right ~85°, drive 3 units with ball detection on
📌The main tuning knob is LAUNCH_RIGHT_TURN — adjust between 83–87 to correct for drift
📌The border sensor is off during the first 3 steps — launchActive = true suppresses it until step 4

// What each stat controls (all computed in computeStats)
currentPower  = SPEED × 100 / 20              // starting cruise speed (SPEED 4 → 20%, SPEED 10 → 50%)
chargePower   = min(POWER², 100)             // POWER 3 → 9, POWER 7 → 49, POWER 10 → 100 — exponential!
FOLLOW_THROUGH_MS = POWER × 200           // how long to push after contact (POWER 3 → 600ms)
thisDuration  = (BASE_SEGMENT_MS × 12) / TURNING  // each arc segment's duration
turnRatio     = 0.2 + TURNING × 0.02        // arc tightness (cap: 0.4)
DECAY_RATE    = 20 − ENDURANCE              // how fast currentPower drops during the match
⚠️
ENDURANCE does not change how long the match runs — it changes how fast your bot slows down. A bot with ENDURANCE 1 has DECAY_RATE 19 and is running at roughly 22% speed by the end. ENDURANCE 17 has DECAY_RATE 3 and barely slows at all.
⚡
POWER scales exponentially. POWER 3 gives chargePower of 9. POWER 7 gives 49. POWER 10 gives 100. Going from 9 to 10 almost doubles your charge. Small increases at the top end matter a lot.
Controls

Two buttons. Simple roles.

A

Start the Match

Validates your stats. If the total is 20, runs computeStats, then the launch sequence, then the match loop. The match runs for 180 seconds.

If stats don't total 20, shows ERR for 1 second and stops. Nothing else runs.

A+B

Emergency Stop

Sets matchRunning = false, stops all motors immediately, and shows an X on the LED. The main loop and the background safety-net both check matchRunning and stop.

computeStats — Turning Stat Points Into Motor Values
Loading blocks from MakeCode…
let SPEED = 0
let ENDURANCE = 0
let TURNING = 0
let POWER = 0
let currentPower = 0
let chargePower = 0
let FOLLOW_THROUGH_MS = 0
let totalArcTime = 0
let BASE_SEGMENT_MS = 0
let thisDuration = 0
let turnRatio = 0
let DECAY_RATE = 0
function computeStats () {
    currentPower = SPEED * 100 / 20
    chargePower = Math.min(POWER * POWER, 100)
    FOLLOW_THROUGH_MS = POWER * 200
    totalArcTime = BASE_SEGMENT_MS * 12
    thisDuration = Math.round(totalArcTime / TURNING)
    turnRatio = 0.2 + TURNING * 0.02
    turnRatio = Math.min(turnRatio, 0.4)
    DECAY_RATE = 20 - ENDURANCE
}

Converts your four stats into the numbers the engine actually uses. Called once after validation, before the launch sequence.

1
currentPower = SPEED × 100 / 20 — Starting motor speed as a percentage. This value decays during the match based on DECAY_RATE.
2
chargePower = min(POWER², 100) — Exponential scaling. POWER 3 → 9. POWER 7 → 49. POWER 10 → 100. Capped so it can't exceed 100% PWM.
3
FOLLOW_THROUGH_MS = POWER × 200 — Duration of the follow-through push after making contact. POWER 3 → 600ms. POWER 7 → 1400ms.
4
thisDuration = (BASE_SEGMENT_MS × 12) / TURNING — Each arc segment's duration. BASE_SEGMENT_MS is 1800ms, so totalArcTime = 21600ms. TURNING 7 → ~3086ms per segment.
5
turnRatio = 0.2 + TURNING × 0.02 (cap 0.4) — How much one motor's power is boosted vs the other during an arc. Higher TURNING = sharper arcs.
6
DECAY_RATE = 20 − ENDURANCE — Plugged into the decay formula every loop iteration. Higher ENDURANCE = lower decay rate = bot stays faster longer.
launchSequence — Zone B: The Pre-Built Opening Move Tune LAUNCH_RIGHT_TURN here
Loading blocks from MakeCode…
let launchActive = false
let currentPower = 0
let LAUNCH_RIGHT_TURN = 0
let msPerUnit = 0
let watchResult = 0
function driveAndWatch(ms: number) {}
function handleBorder() {}
function chargeAndFollowThrough() {}
function launchSequence () {
    launchActive = true
    CutebotPro.setBlockCnt(27, CutebotProDistanceUnits.Cm)
    CutebotPro.trolleySteering(CutebotProTurn.LeftInPlace, 90)
    CutebotPro.pwmCruiseControl(currentPower, currentPower)
    CutebotPro.runBlockCnt(2)
    CutebotPro.stopImmediately(CutebotProMotors.ALL)
    CutebotPro.trolleySteering(CutebotProTurn.RightInPlace, LAUNCH_RIGHT_TURN)
    launchActive = false
    driveAndWatch(msPerUnit * 3)
    CutebotPro.stopImmediately(CutebotProMotors.ALL)
    if (watchResult == 1) {
        handleBorder()
    } else if (watchResult == 2) {
        chargeAndFollowThrough()
    }
}

A four-step opening sequence that positions the bot in the center of the ring before the match loop starts. The border sensor is suppressed during steps 1–3 by launchActive = true.

1
Turn 90° left in place. setBlockCnt(27, Cm) calibrates 1 "block count" = 27 cm. trolleySteering(LeftInPlace, 90) rotates the bot 90° counterclockwise.
2
Drive forward 2 block counts. runBlockCnt(2) drives forward exactly 2 × 27 cm = 54 cm. No border detection yet.
3
Turn ~85° right in place. LAUNCH_RIGHT_TURN (default 85) compensates for leftward drift after step 1. Tune this value 83–87 if the bot doesn't end up facing the arena center.
4
Drive forward 3 more units with detection on. launchActive = false re-enables border and ball checking. driveAndWatch handles anything detected during this final approach.
🎛️
The main tuning knob is LAUNCH_RIGHT_TURN in the constants section. Set to 83 if the bot overshoots right, 87 if it undershoots. Test this without starting a full match first.
matchLoop — The 180-Second Competition Engine
Loading blocks from MakeCode…
let matchRunning = false
let matchTime = 0
let matchStartTime = 0
let motorPower = 0
let SPEED = 0
let DECAY_RATE = 0
let endurancePercent = 0
let enduranceFactor = 0
let currentPower = 0
let lastTurnWasLeft = false
let TURNING = 0
let thisDuration = 0
let watchResult = 0
let consecutiveBorderHits = 0
let ballDetected = false
function endMatch() {}
function driveAndWatch(ms: number) {}
function handleBorder() {}
function handleStuck() {}
function showEnduranceBar(num: number) {}
function matchLoop () {
    while (matchRunning) {
        matchTime = (input.runningTime() - matchStartTime) / 1000
        if (matchTime >= 180) {
            endMatch()
            break
        }
        motorPower = SPEED * 100 / 20
        endurancePercent = 100 - DECAY_RATE * matchTime / 40
        endurancePercent = Math.max(endurancePercent, 20)
        enduranceFactor = endurancePercent / 100
        currentPower = Math.round(motorPower * enduranceFactor)
        currentPower = Math.max(currentPower, 15)
        lastTurnWasLeft = !(lastTurnWasLeft)
        for (let index = 0; index < TURNING; index++) {
            driveAndWatch(thisDuration)
            CutebotPro.stopImmediately(CutebotProMotors.ALL)
            if (watchResult == 1) {
                consecutiveBorderHits += 1
                handleBorder()
                break
            }
            if (watchResult == 2) {
                ballDetected = true
                break
            }
            if (watchResult == 3) {
                handleStuck()
                break
            }
        }
        if (!(ballDetected)) {
            showEnduranceBar(enduranceFactor)
        }
    }
}

The heart of the competition engine. Runs until matchTime ≥ 180 (seconds) or emergency stop. Each pass recalculates power based on time elapsed, then drives one full arc cycle.

1
Endurance decay calculation. endurancePercent = 100 − DECAY_RATE × matchTime / 40. Floored at 20%. Multiplied by starting speed to get currentPower, floored at 15 so motors never stall completely.
2
Arc alternation. lastTurnWasLeft flips each cycle — one cycle arcs left, the next right, creating the S-pattern coverage.
3
Inner loop runs TURNING times. Each pass calls driveAndWatch(thisDuration) for one segment. If an event is detected, the inner loop breaks immediately.
4
Two consecutive border hits flip lastTurnWasLeft automatically to break a corner loop where the bot keeps hitting the same wall.
5
Endurance bar update. After each arc cycle, showEnduranceBar(enduranceFactor) updates the LED — unless the bot just detected a ball (charge sequence is about to run).
driveAndWatch — Drive One Segment, Watch for Events
Loading blocks from MakeCode…
let watchResult = 0
let elapsed = 0
let stuckCount = 0
let BORDER_CHECK_MS = 0
let borderSide = 0
let halfChunk = 0
let remainChunk = 0
let dist = 0
let CONTACT_CM = 0
let BALL_DETECT_CM = 0
let STUCK_HITS = 0
function checkBorder() { return 0 }
function driveAndWatch (ms: number) {
    watchResult = 0
    elapsed = 0
    stuckCount = 0
    while (elapsed < ms) {
        halfChunk = Math.min(BORDER_CHECK_MS, ms - elapsed)
        basic.pause(halfChunk)
        elapsed += halfChunk
        borderSide = checkBorder()
        if (borderSide > 0) {
            CutebotPro.stopImmediately(CutebotProMotors.ALL)
            watchResult = 1
            return
        }
        if (elapsed >= ms) {
            return
        }
        remainChunk = Math.min(BORDER_CHECK_MS, ms - elapsed)
        basic.pause(remainChunk)
        elapsed += remainChunk
        borderSide = checkBorder()
        if (borderSide > 0) {
            CutebotPro.stopImmediately(CutebotProMotors.ALL)
            watchResult = 1
            return
        }
        dist = CutebotPro.ultrasonic(SonarUnit.Centimeters)
        if (dist > CONTACT_CM && dist < BALL_DETECT_CM) {
            CutebotPro.stopImmediately(CutebotProMotors.ALL)
            watchResult = 2
            return
        } else if (dist <= CONTACT_CM) {
            stuckCount += 1
            if (stuckCount >= STUCK_HITS) {
                CutebotPro.stopImmediately(CutebotProMotors.ALL)
                watchResult = 3
                return
            }
        } else {
            stuckCount = 0
        }
    }
}

Drives for the given number of milliseconds while sampling sensors in two passes per 60ms window. Sets watchResult to report what happened.

What watchResult means

0
Clean driveSegment completed normally. No events detected.
1
Border detectedA sensor read above BORDER_THRESHOLD. Motors stopped. handleBorder will back up and pivot.
2
Ball detectedUltrasonic read between CONTACT_CM (5) and BALL_DETECT_CM (40). Motors stopped. Charge sequence runs next.
3
StuckUltrasonic ≤ CONTACT_CM for 3 consecutive reads (~60ms) — physical contact confirmed. handleStuck breaks free.
ℹ️
Border sensors fire when a reading goes ABOVE the threshold — higher sensor values = lighter surface = white border line. A reading of 200+ means the sensor is over the white tape. This is the opposite of what you might expect.
chargeAndFollowThrough — Attack the Ball
Loading blocks from MakeCode…
let chargeHitBorder = false
let chargePower = 0
let enduranceFactor = 0
let borderSide = 0
let dist = 0
let CONTACT_CM = 0
let CHARGE_CHECK_MS = 0
let currentPower = 0
let followStart = 0
let FOLLOW_THROUGH_MS = 0
function checkBorder() { return 0 }
function handleBorder() {}
function chargeAndFollowThrough () {
    chargeHitBorder = false
    let effectiveCharge = Math.round(chargePower * enduranceFactor)
    effectiveCharge = Math.max(effectiveCharge, 15)
    CutebotPro.pwmCruiseControl(effectiveCharge, effectiveCharge)
    let contactCount = 0
    let contactMade = false
    while (!(chargeHitBorder) && !(contactMade)) {
        borderSide = checkBorder()
        if (borderSide > 0) {
            CutebotPro.stopImmediately(CutebotProMotors.ALL)
            chargeHitBorder = true
            break
        }
        dist = CutebotPro.ultrasonic(SonarUnit.Centimeters)
        if (dist <= CONTACT_CM) {
            contactCount += 1
            if (contactCount >= 2) {
                contactMade = true
            }
        } else {
            contactCount = 0
        }
        basic.pause(CHARGE_CHECK_MS)
    }
    CutebotPro.stopImmediately(CutebotProMotors.ALL)
    if (!(chargeHitBorder)) {
        CutebotPro.pwmCruiseControl(currentPower, currentPower)
        followStart = input.runningTime()
        while (input.runningTime() - followStart < FOLLOW_THROUGH_MS) {
            borderSide = checkBorder()
            if (borderSide > 0) {
                CutebotPro.stopImmediately(CutebotProMotors.ALL)
                handleBorder()
                break
            }
            basic.pause(CHARGE_CHECK_MS)
        }
        CutebotPro.stopImmediately(CutebotProMotors.ALL)
        CutebotPro.pwmCruiseControl(-1 * currentPower, -1 * currentPower)
        basic.pause(300)
        CutebotPro.stopImmediately(CutebotProMotors.ALL)
    }
}

Runs when ballDetected = true. Charges at full power, follows through for POWER × 200ms, then backs up. Border-aware throughout — if the bot hits the wall during the charge, it stops and skips the follow-through.

1
Effective charge power is scaled by enduranceFactor. If the bot is at 40% endurance late in the match, charge power drops proportionally. Floored at 15 to prevent stall.
2
Charge loop runs until border or confirmed contact. Two consecutive ultrasonic reads ≤ 5 cm = contact confirmed. Requires 2 reads to filter sensor noise.
3
Follow-through runs at currentPower for FOLLOW_THROUGH_MS. Keeps pushing after contact — how long depends on POWER. Still checks border every 20ms.
4
Backs up 300ms to clear the ball before returning to the main loop.
checkBorder — Read the Line Sensors
Loading blocks from MakeCode…
let sensor1State = 0
let sensor2State = 0
let sensor3State = 0
let sensor4State = 0
let BORDER_THRESHOLD = 0
function checkBorder (): number {
    CutebotPro.trackbitStateValue()
    sensor1State = CutebotPro.trackbitgetGray(TrackbitChannel.One)
    sensor4State = CutebotPro.trackbitgetGray(TrackbitChannel.Four)
    sensor2State = CutebotPro.trackbitgetGray(TrackbitChannel.Two)
    sensor3State = CutebotPro.trackbitgetGray(TrackbitChannel.Three)
    if (sensor1State > BORDER_THRESHOLD) {
        return 1
    } else if (sensor4State > BORDER_THRESHOLD) {
        return 2
    } else if (sensor2State > BORDER_THRESHOLD || sensor3State > BORDER_THRESHOLD) {
        return 3
    }
    return 0
}

Reads all four track sensors. Returns a direction code indicating which side (or center) hit the border. Higher values = lighter surface. Returns 0 if all sensors are on the dark arena mat.

1
Sensor 1 (left outer) above threshold → return 1 (left border). handleBorder will pivot right.
2
Sensor 4 (right outer) above threshold → return 2 (right border). handleBorder will pivot left.
3
Sensor 2 or 3 (inner sensors) above threshold → return 3 (head-on or corner). Triggers the longer backup + bigger pivot in handleBorder.
4
All sensors below threshold → return 0. Clear to continue driving.
handleBorder — Back Up and Pivot Away
Loading blocks from MakeCode…
let isRoaming = false
let BORDER_PIVOT_POWER = 0
let BACKUP_MS = 0
let borderSide = 0
let BORDER_PIVOT_MS = 0
let lastTurnWasLeft = false
let recheck = 0
function checkBorder() { return 0 }
function handleBorder () {
    CutebotPro.stopImmediately(CutebotProMotors.ALL)
    isRoaming = false
    basic.pause(50)
    CutebotPro.pwmCruiseControl(-1 * BORDER_PIVOT_POWER, -1 * BORDER_PIVOT_POWER)
    basic.pause(BACKUP_MS)
    CutebotPro.stopImmediately(CutebotProMotors.ALL)
    basic.pause(100)
    if (borderSide == 1) {
        CutebotPro.pwmCruiseControl(BORDER_PIVOT_POWER, -1 * BORDER_PIVOT_POWER)
        basic.pause(BORDER_PIVOT_MS + Math.randomRange(-80, 80))
        lastTurnWasLeft = false
    } else if (borderSide == 2) {
        CutebotPro.pwmCruiseControl(-1 * BORDER_PIVOT_POWER, BORDER_PIVOT_POWER)
        basic.pause(BORDER_PIVOT_MS + Math.randomRange(-80, 80))
        lastTurnWasLeft = true
    } else {
        CutebotPro.pwmCruiseControl(-1 * BORDER_PIVOT_POWER, -1 * BORDER_PIVOT_POWER)
        basic.pause(BACKUP_MS + 300)
        CutebotPro.stopImmediately(CutebotProMotors.ALL)
        basic.pause(100)
        CutebotPro.pwmCruiseControl(BORDER_PIVOT_POWER, -1 * BORDER_PIVOT_POWER)
        basic.pause(BORDER_PIVOT_MS * 2)
        lastTurnWasLeft = false
    }
    CutebotPro.stopImmediately(CutebotProMotors.ALL)
    basic.pause(100)
    recheck = checkBorder()
    if (recheck > 0) {
        borderSide = recheck
        CutebotPro.pwmCruiseControl(-1 * BORDER_PIVOT_POWER, -1 * BORDER_PIVOT_POWER)
        basic.pause(BACKUP_MS)
        CutebotPro.stopImmediately(CutebotProMotors.ALL)
        basic.pause(100)
        CutebotPro.pwmCruiseControl(-1 * BORDER_PIVOT_POWER, BORDER_PIVOT_POWER)
        basic.pause(BORDER_PIVOT_MS)
        CutebotPro.stopImmediately(CutebotProMotors.ALL)
        basic.pause(100)
    }
    isRoaming = true
}

Called when borderSide > 0. Direction of the pivot depends on which side triggered. A small random offset prevents the bot from always pivoting the same amount and getting stuck in a corner loop.

1
Sets isRoaming = false — disables the background border checker while recovery is in progress.
2
Reverses at BORDER_PIVOT_POWER for BACKUP_MS (600ms) to clear the sensor from the white line.
3
borderSide 1 = left → pivot right. borderSide 2 = right → pivot left. borderSide 3 = head-on → extra backup + double-length pivot.
4
Rechecks with checkBorder() after pivoting. If still on the border, backs up and pivots again — safety net for deep incursions.
5
Sets isRoaming = true — re-enables the background border checker before returning.

function handleStuck () {
    CutebotPro.stopImmediately(CutebotProMotors.ALL)
    isRoaming = false
    basic.showLeds(`
        # . . . #
        . # . # .
        . . # . .
        . # . # .

        lastTurnWasLeft = true
    }
    basic.pause(STUCK_PIVOT_MS)
    CutebotPro.stopImmediately(CutebotProMotors.ALL)
    basic.pause(100)
    isRoaming = true
}

Called when watchResult = 3 — the ultrasonic confirmed physical contact for 3 consecutive reads. Shows an X on the LED, backs up, and pivots in the opposite direction from last time to avoid spinning in place.

1
Shows an X on the LED so you know a stuck recovery is happening.
2
Backs up at BORDER_PIVOT_POWER for BACKUP_MS (600ms).
3
Pivot direction alternates based on lastTurnWasLeft — prevents the bot from always escaping the same way and re-getting stuck.
4
Pivots for STUCK_PIVOT_MS (600ms) — slightly longer than a normal border pivot for a stronger escape.

forever — Background Border Safety Net

let matchRunning = false
let isRoaming = false
let launchActive = false
let borderSide = 0

basic.forever(function () {

Runs independently from the main match loop, continuously in the background. While roaming, it checks the border sensors every 30ms — twice as fast as the 60ms check inside driveAndWatch.

This is the safety net. If the bot ever crosses the border during a moment when driveAndWatch is between checks, this loop catches it. It only runs when matchRunning is true, isRoaming is true, and launchActive is false — so it doesn't interfere with the launch sequence or recovery moves.

Strategy Tips

Getting more out of the match code

01

Test your stats first

Run the Endurance Test and Turning Test with your actual stat values before competition. The match code uses the same decay and arc formulas — surprises in testing are better than surprises in a match.

02

ENDURANCE ≠ match length

All matches run 180 seconds. ENDURANCE controls how fast your speed drops. ENDURANCE 1 (DECAY_RATE 19) reaches ~22% speed by the end. ENDURANCE 17 (DECAY_RATE 3) barely slows at all.

03

POWER scales exponentially

Going from POWER 3 (chargePower 9) to POWER 4 (chargePower 16) is nearly doubling your charge force. The same stat point at POWER 9 → 10 takes you from 81 to 100. Where you put POWER matters a lot.

04

Tune LAUNCH_RIGHT_TURN early

Test the launch sequence before competition and adjust the constant if needed. Overshoot right = lower the value toward 83. Undershoot = raise it toward 87. A well-aimed launch means less time correcting position.

05

Watch the LED for clues

The center-column endurance bar shows power remaining. "C" = charging. "F" = follow-through. An X = stuck recovery. If you see lots of X patterns, the bot may be running into walls frequently.

06

ERR = check your math

If you press A and see ERR, add up your four stats manually. A common mistake: adjusting one stat without adjusting another, so the total drifts past 20. Exact 20 required — no exceptions.
